Overview

This tool mentor assumes that a  User-Experience Model has been created using Setting Up User Experience Modeling for a Project Using Rational Rose.

This tool mentor assumes the following:

  • Use-Case Model has been created using Tool Mentor: Setting Up Rational Rose for a Project.
  • The Use-Case Model has been populated with Actor.
  • Use Cases by following Tool Mentor: Finding Actors and Use Cases Using Rational Rose.

The following are performed in this tool mentor:

  • Modeling User-Experience Elements
  • Modeling User-Experience Storyboards
  • Modeling the User-Experience Navigation Map

References to Rose Help topics are marked with .

For more information on modeling the User-Experience Model, see Guideline: User Experience Modeling .

Modeling User-Experience Elements

Capture new  User-Experience Elements as follows:

  1. Navigate to the User-Experience Model package that is to contain the User-Experience Elements. For more information on the structure of the User-Experience Model, see  Guideline: User-Experience Model.
  2. If necessary, add class diagrams as needed to capture the User-Experience Elements.
  3. Add new User-Experience Elements as they are identified, using the proper stereotypes (see  Guideline: User-Experience Element).

Describe each of the User-Experience Elements as follows:

  1. Add a description for each User-Experience Element class in the class documentation field.
  2. Enter actions that can be performed on the User-Experience Element as operations of the User-Experience Element class.
  3. Enter content to be displayed on the User-Experience Element as attributes of the User-Experience Element class.
  4. (optional) Associate a document with the class. This document could be a screen shot from an user-interface prototyping tool.

For more information about creating and documenting classes, refer to the Class topic in the Rational Rose online help.

For more information on modeling User-Experience Elements, see  Guideline: User-Experience Element.

Modeling User-Experience Storyboards

The following sections describe how to model a  User-Experience Storyboard:

  • Creating a User-Experience Storyboard
  • Modeling a User-Experience Storyboard Participants Diagram
  • Modeling the User-Experience Storyboard Flows of Events
  • Modeling User-Experience Storyboard Flow Diagrams

For more information on modeling storyboards, see  Guideline: User-Experience Storyboard.

Creating a User-Experience Storyboard

  1. Navigate to the User-Experience Model package where  User-Experience Storyboards are to be created. For more information on the structure of the User-Experience Model, see  Guideline: User-Experience Model.
  2. Create a use-case diagram to capture the User-Experience Storyboards (unless one already exists).
  3. Add a Use Case model element (the User-Experience Storyboard) to the diagram. Assign it the same name as the Use Case it realizes. Assign it a stereotype of <<use-case storyboard>>. If a dialog appears indicating the use case exists in two name spaces, click OK. This new use case is the use-case storyboard.
  4. Drag and drop the Use Case being realized onto the diagram.
  5. Add a realization relationship from the User-Experience Storyboard to the Use Case.

Modeling a User-Experience Storyboard Participants Diagram

Capture the participants (User-Experience Elements) in the User-Experience Storyboard as follows:

  1. Navigate to the User-Experience Storyboard, and create an associated Participants Diagram (a class diagram).
    Right click on the User-Experience Storyboard and select "New>>Class Diagram". Name the diagram "Participants".
    It is also helpful to insert the diagram title on the top of the diagram. Title suggestion: <use case name> User-Experience Storyboard Participants".
  2. Drag and drop Actors and the previously identified participating User-Experience Elements onto the diagram.
  3. Adjust the level of detail displayed for the User-Experience Elements.  See  Guideline: User-Experience Storyboard for guidance on the level of detail that should be displayed in the participant diagram

Define the navigation paths between the User-Experience Elements as follows:

  1. On the participants diagram, add association relationships between User-Experience Elements to represent the navigation. The navigation associations can be created using the unidirectional association tool from the Rose tool bar.
  2. Name and provide a brief description of the navigation.

For more information on modeling User-Experience Elements in Rational XDE, see the Modeling User-Experience Elements section of this tool mentor.

Modeling the User-Experience Storyboard Flows of Events

Describe the User-Experience Storyboard's flows of events in terms of the participating User-Experience Elements. The resulting detailed flow of events can be captured in a separate document "attached" to the User-Experience Storyboard, or the detailed flows of events can be included as a script on the User-Experience Storyboard flow diagrams (see the Modeling User-Experience Storyboard Flow Diagrams section).

Modeling User-Experience Storyboard Flow Diagrams

The User-Experience Storyboard flow diagrams can be captured in Rational Rose using either Collaboration Diagrams or Sequence Diagrams. Collaboration diagrams tend to be easier to draw on a white-board, while Sequence diagrams portray object interactions and time-sequencing in a more intuitive way. The choice of which one to use is largely a matter of taste and project preferences.

No matter which diagram type you choose, the following must be performed:

  1. Create the diagram under the User-Experience Storyboard (right click on the User-Experience Storyboard and select "New").
    It is helpful to insert the diagram title on the top of the diagram.  Title suggestion: <use case name> User-Experience Storyboard: <flow name>".
  2. (optional) Enter a brief description of the flow of events that the sequence diagram depicts.
  3. Drag and drop the participating Actors and User-Experience Elements onto the interaction diagram.
  4. Add messages between the objects.
  5. (optional) Describe the message.
  6. To describe how object behaves when it receives the message, assign an operation to the message (right click on the message and select the action from the drop-down list).
  7. If the flow between the User-Experience Elements is not clear from the messages, use notes to annotate the diagram.
    Scripts can be entered via the free form text tool on the tool bar and the text can be associated to a message by selecting the text and the message and selecting Edit>>Attach Script.
    The note icon is also available on the tool bar.

For information on sequence and collaboration diagrams, see: 

  • Sequence Diagrams (Overview) topic in the Rational Rose online help.

  • Collaboration Diagrams (Overview) topic in the Rational Rose online help.

  • Chapter 4, Introduction to Diagrams and Chapter 9, Interaction Diagrams and Specifications in the Using Rational Rose manual.


Modeling the User-Experience Navigation Map

If a  User-Experience Navigation Map has not been created yet, perform the following steps:

  1. Open the User-Experience Model.
  2. Navigate to the package or packages that are to contain navigation map diagram(s). For recommendations on where in the User-Experience Model structure to create navigation map diagrams, see  Guideline: User-Experience Model.
  3. Add a class diagram(s) to document the navigation map.

Populate the navigation map as follows:

  1. Open the User-Experience Model.
  2. Navigate to the class diagram that represents the navigation map.
  3. Drag and drop the User-Experience Elements on to the diagram. Rose will automatically draw any existing navigation relationships that exist between the elements as they are added to the diagram.
    This assumes that the User-Experience Elements already exist. For information on modeling the User-Experience Elements, see Modeling the User-Experience Elements.

  4. If the navigation associations between the User-Experience Model Elements were created in the User-Experience Storyboard as described above in Modeling the User-Experience Storyboard, the associations will be placed in the User-Experience Storyboard's package.  The navigation associations really should be placed with the User-Experience Element that initiates the navigation.  Thus, relocate the associations from the use-case storyboard's package to the appropriate Ux Element screen package.

    For more information about relocating associations, refer to the Moving Classes, Packages or Associations Between Tiers topic in the Rational Rose online help. 

Capture the navigation associations between the User-Experience Elements as follows:

  1. Define any needed additional navigation paths between the User-Experience Elements by adding directed association relationships between the User-Experience Elements to represent the navigation.
  2. Name and provide a brief description of the navigation.

To reduce clutter on the navigation map diagrams, you can reduce the amount of information displayed for each of the User-Experience Elements. For example, you could just include the User-Experience Element, its name, and the navigation associations, and not include the User-Experience Element content (attributes) or actions (operations). See  Guideline: User-Experience Navigation Map for guidance on the level of detail that should be displayed in the navigation map diagram.
