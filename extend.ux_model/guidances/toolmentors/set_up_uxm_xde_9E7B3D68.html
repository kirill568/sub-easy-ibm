Руководство по инструменту: Setting Up User Experience Modeling for a Project Using Rational XDE Developer
This tool mentor describes how to set up Rational XDE™ to include a User-Experience Model, as well as the how to install the user-experience stereotypes and icons.

The steps for setting up user-experience modeling in XDE are described below:

  • Install the XDE Support Files
  • Create the User-Experience Model
  • Associate the User-Experience Stereotype Icons

Install the XDE Support Files

To support user-experience modeling in XDE, a set of support files have been provided. The XDE support files include an XDE model template for the User-Experience Model that reflects the User-Experience Model packaging structure described in  Guideline: User-Experience Model, as well as customized icons for the user-experience modeling stereotypes.

The following steps should be performed in order to install these support files:

  1. Unzip the zip file containing the XDE support files for the User-Experience Modeling plug-in (uxm_support_xde.zip) into some directory
    (Suggestion: Create a subdirectory called "UX Modeling" in the <XDE install directory/Addins> directory).
    Two subdirectories will be created: "templates", which contains the XDE model template files, and "icons", which contains the user-experience modeling stereotype icons.
  2. Move the User-Experience Model XDE model template files (all files in the templates subdirectory) to the XDE template directory. The XDE template directory is different depending on what platform you are executing XDE on.
    • For XDE for IBM WSSAD, the template directory is: <directory where WSSAD was installed>\plugins\com.rational.hshell.ratltool\templates\Rational XDE.
      For example: C:\Program Files\IBM\Application Developer\plugins\com.rational.hshell.ratltool\templates\Rational XDE.
    • For XDE for Microsoft®  VS .NET, the template directory is: <directory where XDE was installed>\XDEPkg\Rational XDE.
      For example: C:\Program Files\Rational\XDE\XDEPkg\Rational XDE.
  3. You can delete the templates directory as it should now be empty.

To confirm that the templates installed correctly, perform the following steps:

  1. Launch XDE.
  2. Right click a project. 
    If you are using XDE for IBM WSSAD, select New>Model.
    If you are using XDE for Microsoft®  VS .NET, Add>Add New Item.
  3. In the File Types list, select Rational XDE.
    "User-Experience Model" should be included in the list of templates.

You can further specialize this downloaded template to reflect your project-specific guidelines. Rational XDE lets you set operating defaults, called properties, and options that essentially establish the "rules" to follow when you are working with the XDE model. This includes default fonts, colors, line styles, file storage options, the ways in which which stereotypes and other information are presented, and the profiles that are applied.

Project-specific defaults can be defined for the User-Experience Model by editing the provided User-Experience Model template, setting its defaults, and exporting it as a model template.

Create the User-Experience Model

In this step, you create the XDE model file that will contain the User-Experience Model, using the previously installed XDE model template and stereotypes.

The User-Experience Model is represented in an XDE "Blank Model" model file named "User-Experience Model".

In XDE, all models must exist with a project (where "project" is defined by the underlying IDE platform). It is recommended that the XDE model file containing the User Experience Model exist within a project that represents the entire application.

Perform the following steps to create the User-Experience Model in XDE.

  1. Right click your project (usually the application project) and select New>>Model.
  2. In the File Types list, select "Rational XDE".
  3. In the model template list, select "User-Experience Model".
  4. An XDE model will be created and opened.

The created User-Experience Model file contains some sample packages, User-Experience Elements (<<screen>> and <<input form>> classes) and User-Experience Storyboards (<<storyboard>> collaborations). These elements should be replaced with elements for the application being developed. If special stereotype icons are not displayed for the User-Experience Elements, then perform the steps in the next section, Associate the User-Experience Stereotype Icons.

Associate the User-Experience Stereotype Icons

The steps in this section only need to be performed if special stereotype icons are not displayed for the sample User-Experience Elements in the User-Experience Model file created in the previous section.

To associate the installed icons with the installed user-experience stereotypes, perform the following steps for each stereotype:

  1. Select a model element with that stereotype.
  2. From the stereotype properties dialog box, set the explorer and diagram icons to the appropriate installed graphics files (see table below).

The explorer and diagram icons for each of the user-experience modeling stereotypes are listed in the following table (all pathnames are relevant to the directory where you unzipped the user-experience modeling XDE support files):

Stereotype Explorer Icon Diagram Icon
<<screen>> icons/list/screen.bmp icons/diagram/screen.emf
<<input form>> icons/list/screen-input-form.bmp icons/diagram/screen-input-form.emf
