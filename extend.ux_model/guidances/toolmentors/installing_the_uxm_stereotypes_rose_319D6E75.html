Руководство по инструменту: Setting Up User Experience Modeling for a Project Using Rational Rose
This tool mentor describes how to set up Rational Rose™ to include a User-Experience Model, as well as the how to install the user-experience stereotypes and icons.
Инструмент: Rational Rose

Основное описание

The following topics are discussed in this tool mentor.

  • Install the XDE Support Files
  • Create the User-Experience Model

Install the XDE Support Files

To support user-experience modeling in Rose, a set of custom stereotypes and icons have been provided. These must be installed into Rose prior to modeling the user-experience elements in Rose.

The stereotypes and their associated icons are as follows:

  • <<screen>> class stereotype

  • <<input form>> class stereotype

Tool Steps

  1. Unzip the installation wizard zip file into /temp.  An executable should be extracted.
  2. Run the executable
  3. Re-start Rose
  4. Create a class diagram containing a class with the stereotype, <<screen>>.  Right click the new class and select "Options>>Stereotype Display>>Icon".  The icon displayed should be just like the one shown above.
  5. Create a class diagram containing a class with the stereotype, <<input form>>.  Right click the new class and select "Options>>Stereotype Display>>Icon".  The icon displayed should be just like the one shown above.

Create the User-Experience Model

In this step, you create the Rose model file that will contain the User-Experience Model, using the previously installed stereotypes.

The  Work Product: User-Experience Model can be represented in Rational Rose using a package within the Logical View named "User-Experience Model".

Within the created User-Experience Model, create packages to reflect the recommended model structure, as described in  Guideline: User-Experience Model.

For more information about creating packages, refer to the Package topic in the Rational Rose online help.
