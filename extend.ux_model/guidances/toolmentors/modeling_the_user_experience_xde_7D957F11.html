Основное описание

This tool mentor assumes that a  User-Experience Model has been created using Tool Mentor: Setting Up User Experience Modeling for a Project Using Rational XDE Developer. It also assumes that Actors have already been created (see Tool Mentor: Finding Actors and Use Cases Using Rational XDE Developer).

The following are performed in this tool mentor:

  • Modeling User-Experience Elements
  • Modeling User-Experience Storyboards
  • Modeling the User-Experience Navigation Map

Links to topics in the Rational XDE™ online help are marked with .

For more information on modeling the User-Experience Model, see Guideline: Modeling the User Experience.

Modeling User-Experience Elements

Capture new  User-Experience Element as follows:

  1. Navigate to the User-Experience Model package that is to contain the User-Experience Elements. For more information on the structure of the User-Experience Model, see  Guideline: User-Experience Model.
  2. If necessary, add class diagrams as needed to capture the User-Experience Elements. See .
  3. Add new User-Experience Elements as they are identified. See .

Describe each of the User-Experience Elements as follows:

  1. Add a description to each User-Experience Element class. See .
  2. Enter actions that can be performed on the User-Experience Element as operations of the User-Experience Element class. See .
  3. Enter content to be displayed on the User-Experience Element as attributes of the User-Experience Element class. See .
  4. (optional) Associate a document with the class. See . This document could be a screen shot from an user-interface prototyping tool.

For more information on modeling User-Experience Elements, see  Guideline: User-Experience Element.

Modeling User-Experience Storyboards

The following sections describe how to model a  User-Experience Storyboard:

  • Creating a User-Experience Storyboard
  • Modeling a User-Experience Storyboard Participants Diagram
  • Modeling the User-Experience Storyboard Flows of Events

For more information on modeling storyboards, see  Guideline: User-Experience Storyboard.

Creating a User-Experience Storyboard

  1. Navigate to the User-Experience Model package where  User-Experience Storyboards are to be created. For more information on the structure of the User-Experience Model, see  Guideline: User-Experience Model.
  2. Create a use-case diagram to capture the User-Experience Storyboards (unless one already exists). See .
  3. Add a collaboration instance (the User-Experience Storyboard) to the diagram. Assign it the same name as the Use Case it realizes. Assign it a stereotype of <<use-case storyboard>>. Reference: , , .
  4. Drag and drop the Use Case onto the diagram. See .
  5. Add a realization relationship to the Use Case. See .

Modeling a User-Experience Storyboard Participants Diagram

Capture the participants (User-Experience Elements) in the User-Experience Storyboard as follows:

  1. Navigate to the User-Experience Storyboard, and create an associated Participants Diagram (a class diagram). See .
  2. Drag and drop Actors and the previously identified participating User-Experience Elements onto the diagram. See .

Define the navigation paths between the User-Experience Elements as follows:

  1. On the participants diagram, add association relationships between User-Experience Elements to represent the navigation. See .
  2. Name and provide a brief description of the navigation. See .

For more information on modeling User-Experience Elements in Rational XDE, see the Modeling User-Experience Elements section of this tool mentor.

Modeling the User-Experience Storyboard Flows of Events

Describe the User-Experience Storyboard's flows of events in terms of the participating User-Experience Elements. The resulting detailed flow of events can be captured in a separate document "attached" to the User-Experience Storyboard (see ), or the detailed flows of events can be included as a script on the User-Experience Storyboard flow diagrams (see the Modeling User-Experience Storyboard Flow Diagrams section).

Modeling User-Experience Storyboard Flow Diagrams

  1. For each User-Experience Storyboard flow of events create one or more interaction instances. (In the Model Explorer, right-click the User-Experience Storyboard, click Add UML, and then click on Interaction Instance.)

  2. Create a sequence diagram for this interaction instance. See .

  3. (optional) Enter a brief description of the flow of events that the sequence diagram depicts. See .
  4. Drag and drop the participating Actors and User-Experience Elements onto the sequence diagram. See .
  5. Add messages between the objects. See .
  6. (optional) Describe the message. See .
  7. To describe how object behaves when it receives the message, assign an operation to the message. See .
  8. If the flow between the User-Experience Elements is not clear from the messages, use notes to annotate the diagram. See .

Modeling the User-Experience Navigation Map

If a  User-Experience Navigation Map has not been created yet, perform the following steps:

  1. Open the User-Experience Model.
  2. Navigate to the package or packages that are to contain navigation map diagram(s). For recommendations on where in the User-Experience Model structure to create navigation map diagrams, see  Guideline: User-Experience Model.
  3. Add a class diagram(s) to document the navigation map. See .

The user interface elements are captured as  User-Experience Elements in the navigation map as follows:

  1. Open the User-Experience Model.
  2. Navigate to the class diagram that represents the navigation map.
  3. Add classes to the diagram, stereotyped as <<screen>> or <<input form>>. Reference: , .
  4. Add a description to each class. See .
  5. (optional) Associate a document with the class. See . This document could be a bitmap from an user-interface prototyping tool, a screen mock-up, etc.

Capture the navigation associations between the User-Experience Elements as follows:

  1. Define navigation paths between the User-Experience Elements by adding directed association relationships between the User-Experience Elements to represent the navigation. See .
  2. Name and provide a brief description of the navigation. See .

To reduce clutter on the navigation map diagrams, you can reduce the amount of information displayed for each of the User-Experience Elements. For example, you could just include the User-Experience Element, its name, and the navigation associations, and not include the User-Experience Element content (attributes) or actions (operations).
