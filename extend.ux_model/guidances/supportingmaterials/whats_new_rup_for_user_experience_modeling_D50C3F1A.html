What's New for RUP for User-Experience Modeling
Основное описание

The new features and changes from version to version are described below.

  • From 3.1 to 4.0
  • From 3.0 to 3.1
  • From 2.0 to 3.0

From 3.1 to 4.0

The following were the key changes in this release:

  • Migrated content to the Rational Method Composer (RMC) platform. 
  • Eliminated many of the individual guidelines pages.  RMC supports method content contribution, so much of the content originally captured in separate guidelines associated with method content (e.g., tasks, work products, etc.) could be added to the original via contribution.
  • Merged separate XDE tool mentors for the Java and .NET platforms.  Any differences (minor) are noted inline.  This is consistent with what RUP has done for the XDE tool mentors,

From 3.0 to 3.1

Minor copyedits only.

From 2.0 to 3.0

Release 3.0 changes consist primarily of look and feel and structure changes to accommodate changes to the underlying RUP process.

Also, some content originally in the User-Experience Modeling plug-in have been moved into base RUP (e.g., the Navigation Map).

The scope of the User-Experience Modeling has been adjusted to just focus on the production of the User-Experience Model.
