Check the Consistency between the Analysis Classes and the User-Experience Elements

It is important to check the consistency between the Analysis Classes and the  User-Experience Elements in the  User-Experience Model because the User-Experience Model is the contract between the presentation logic development teams and the business logic development teams.  Thus, it is important that the presentation requirements and the business aspects of the system are synchronized.

  • Every boundary Analysis Class in the Analysis Model that represents a user interface should have at least one associated User-Experience Element in the User-Experience Model.
  • The dynamic content identified for the User-Experience Elements should be provided by the business logic.
  • The user actions identified for the User-Experience Elements (and any user-supplied content identified for input forms should be processed by the business logic.
