Руководство по инструменту: Развертывание Web-приложения на WebSphere Application Server
В этой памятке по инструменту приведены инструкции по развертыванию Web-приложений на сервере WebSphere Application Server с помощью среды моделирования RAD 6.0.

Обзор

Перед развертыванием Web-приложения скомпонуйте его в архивный файл ( Модуль J2EE или  Приложение J2EE). После этого для развертывания приложения нужно будет только установить архивный файл на сервере с помощью инструментов администрирования WebSphere. В этой памятке по инструменту предусмотрены следующие шаги:

  • Создание архивного файла приложения
  • Установка архивного файла на сервере WebSphere Application Server

Создание архивного файла приложения

Если приложение было разработано в RAD 6.0, то для создания архивного файла выполните инструкции из раздела Памятка по инструменту: Компоновка модулей и приложений J2EE с помощью RAD 6.0. Для создания архивных файлов других приложений используйте инструментарий Assembly toolkit, который входит в WebSphere Application Server. Для этого выполните следующие шаги:

  1. Запустите Assembly Toolkit.
    1. В Windows выберите Пуск > Программы > IBM > ASTK > ASTK.
    2. В окне Application Server Toolkit укажите имя Рабочей области и нажмите OK.
  2. Для создания файла EAR (Enterprise Application Archive) выполните следующие действия:
    1. В проекции J2EE выберите Файл > Создать > Проект приложения J2EE.
    2. С помощью мастера Создания проекта приложения J2EE выберите уровень J2EE, укажите имя и расположение файла EAR и выберите, какие модули нужно включить в файл EAR (Web, EJB, приложение-клиент или коннектор).
  3. Для создания файла Web-архива (WAR) выполните следующие действия:
    1. В проекции J2EE выберите Файл > Создать > Динамический Web-проект.
    2. С помощью мастера Создания Web-проектов укажите имя и расположение файла WAR, свяжите его с файлом EAR и задайте значения для опций Web-проекта.
    3. После создания Web-проекта можно импортировать Web-содержимое приложения с помощью мастера Импорта. Мастер импорта можно запустить из меню Файл.
  4. Для создания файла EJB JAR выполните следующие действия:
    1. В проекции J2EE выберите Файл > Создать > Проект EJB.
    2. С помощью мастера Создания проектов EJB выберите версию спецификации EJB, укажите имя и расположение файла JAR, свяжите его с файлом EAR и задайте зависимости между модулями и файлом JAR.
    3. После создания проекта EJB можно импортировать объекты EJB и связанные продукты работы кода с помощью мастера Импорта. Мастер импорта можно запустить из меню Файл.
  5. Для создания файла JAR приложения-клиента выполните следующие действия:
    1. В проекции J2EE выберите Файл > Создать > Проект приложения-клиента.
    2. С помощью мастера Создания проекта приложения-клиента выберите уровень J2EE, укажите имя и расположение файла JAR, свяжите его с файлом EAR и задайте зависимости между модулями и файлом JAR.
  6. Для создания файла RAR адаптера ресурса выполните следующие действия:
    1. В проекции J2EE выберите Файл > Создать > Проект коннектора.
    2. С помощью мастера Создания проекта коннектора укажите имя и расположение файла JAR, свяжите его с файлом EAR и задайте зависимости между модулями и файлом JAR.
  7. При необходимости отредактируйте файлы описания модулей приложений-клиентов, EJB, Web или приложений J2EE.
    1. В панели Иерархия J2EE щелкните правой кнопкой мыши на модуле и выберите Открыть с помощью > Редактор файлов описания.
    2. Для изменения значений параметров файла описания выберите соответствующую вкладку в нижней части панели редактора.
  8. Сгенерируйте код развертывания для модулей EJB.
    1. В панели Иерархия J2EE щелкните правой кнопкой мыши на модуле и выберите Открыть с помощью > Код развертывания и RMIC.
    2. Убедитесь, что во время создания кода не появлялись сообщения об ошибках.
  9. Сгенерируйте код развертывания для модулей Web-служб.
    1. В панели Иерархия J2EE щелкните правой кнопкой мыши на модуле и выберите Web-службы > Развернуть Web-службу.
    2. Убедитесь, что во время создания кода не появлялись сообщения об ошибках.

Установка архивного файла на сервере WebSphere Application Server

По завершении компоновки приложения разверните файлы EAR, WAR или JAR на сервере приложений с помощью инструментов администрирования WebSphere. Для GUI используйте Административную консоль, а для интерфейса на базе сценариев - утилиту wsadmin. Для работы с Административной консолью выполните следующие действия:

  1. Запустите сервер приложений. Для этого в командной строке введите команду startServer.
  2. Запустите административную консоль. Для этого откройте Web-браузер и введите адрес http:// имя_сервера:9090/admin.
  3. Войдите в консоль.
  4. Выполните инструкции по установке нового приложения.
